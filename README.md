# 업무관리 Decision Journey MVP v0.3.2.2

팀장급 리더 교육을 위한 업무관리 딜레마 Decision Journey 웹앱입니다.

## v0.3.2.2에서 달라진 점

- 교육생 입력값을 브라우저 임시 저장소(localStorage)에 저장합니다.
- 라운드 진행 화면마다 하단에 `이전 / 홈 / 다음` 이동 버튼을 추가했습니다.
- `홈` 버튼을 누르면 언제든지 딜레마 라운드 선택 페이지로 이동합니다.
- 1차 판단과 2차 판단 화면에서는 선택값이 없으면 하단 `다음` 버튼이 비활성화됩니다.
- 같은 브라우저에서 `/instructor`로 이동하면 저장된 실제 입력값이 강사용 대시보드에 반영됩니다.
- 10라운드 전체에 실제 업무관리 딜레마 콘텐츠를 넣었습니다.
- 아직 Google Sheets API와 Gemini API는 연결하지 않았습니다.

> 주의: v0.3.2.2의 저장은 브라우저 localStorage 기반입니다. 즉, 교육생 스마트폰과 강사용 PC가 서로 다른 기기이면 데이터가 자동 공유되지 않습니다. 다음 단계에서 Google Sheets API를 연결하면 여러 기기 간 데이터 공유가 가능합니다.

## 화면 경로

| 경로 | 화면 |
|---|---|
| `/` | 교육생용 모바일 화면 |
| `/instructor` | 강사용 PC 대시보드 |

## 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:3000
http://localhost:3000/instructor
```

## Vercel 배포

1. GitHub에 프로젝트 파일을 업로드합니다.
2. Vercel에서 해당 GitHub Repository를 Import합니다.
3. Framework는 Next.js로 자동 인식됩니다.
4. Deploy를 클릭합니다.

## v0.3.2.2 테스트 방법

1. `/`에서 세션 코드와 팀명을 입력합니다.
2. 라운드를 선택합니다.
3. 1차 판단, 돌발상황, 2차 판단, AI 질문, 최종 5줄 결정문을 입력합니다.
4. “제출하고 저장하기”를 누릅니다.
5. 같은 브라우저에서 `/instructor`로 이동합니다.
6. 강사용 대시보드에 실제 입력값이 반영되는지 확인합니다.
